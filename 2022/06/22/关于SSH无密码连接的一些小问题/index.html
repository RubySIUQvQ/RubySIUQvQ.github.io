关于SSH无密码连接的一些小问题

发表于2022-06-22|更新于2022-06-25

SSH是远程连接的常用工具，通常为了方便都会通过分发公钥的方式来实现无密码的登录，但首次配置无密码登陆时可能由于自己对SSH的不熟悉而遇到一些无法预料的问题，特此将首次配置所遇到的问题记录并给出解决方案。

本文所记录的均是网络上的配置教程中大概率不会提及的问题。

1. SSH的登录名

我们使用SSH连接远程主机时如果不指定登录名，则默认使用本机当前用户登录。

同时也可以在连接时指定登录名。

以上是SSH的基本使用语法，但是在采用无密码登录时会遇到这样一种问题，即我们生成密钥时，使用的是我们本机登录的用户名，而且生成密钥的用户名无法指定，只能使用当前用户。这就导致我们无法采用无密码的方式登录到不同名用户，此问题尚未找到完美的解决方法，只能使用相同的用户名来创建密钥。以下给出Linux新建用户和Windows修改用户名的方法。

Linux
Windows

在左下角开始面板处右键，打开计算机管理选项（或直接搜索计算机管理），进入后在左侧选择本地用户和组，在用户中找到要修改的用户，右键即可重命名。完成此修改后whoami指令将显示修改后的名称。

2. SSH目录的权限问题

如果采用传输公钥并手动生成authorized_keys文件的方式来分发公钥，可能会出现SSH目录权限不合要求的问题。

SSH要求

  1. 用户目录权限为755或700，即其他用户不能有写权限
  2. .ssh目录权限为755或700
  3. rsa_id.pub以及authorized_keys文件权限为644
  4. ras_id权限必须为600

通常来说.ssh以及密钥文件为自动生成，不会出现权限问题，但authorized_keys可能由用户手动创建，如果该文件权限不正确，SSH不会识别其中的公钥，导致出现分发了公钥但仍然需要密码登录的问题。

修改文件权限或者使用指令分发公钥即可解决该问题
